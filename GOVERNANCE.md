# 谛听治理文档

## 角色切片命名规范

### 命名格式

```
[角色名]-[切片描述]-[版本]
```

示例：
- `wukong-journey` - 取经路上的悟空
- `wukong-naive` - 大闹天宫的悟空
- `lindiya-youth` - 少女时期的林黛玉
- `zhanggong-idealism` - 理想主义时期的张飞

### 切片命名原则

1. **描述性**：切片名应描述该时期的核心特征
2. **简洁**：2-3个单词内
3. **唯一性**：同一角色的不同切片应可区分

## 流程检查点规范

### Phase 1.5 调研Review检查点

所有Agent完成后，必须暂停展示调研质量摘要：

```
┌──────────────────┬──────────┬──────────────────────────┐
│ Agent            │ 来源数量  │ 关键发现                  │
├──────────────────┼──────────┼──────────────────────────┤
│ 1 原著文本       │ X处      │ 核心场景: ...            │
│ 2 表演风格       │ X处      │ 标志性动作: ...          │
│ 3 关系网络       │ X条      │ 主要关系: ...           │
│ 4 成长弧线       │ X阶段    │ 关键转折: ...           │
│ 5 同人/分析      │ X篇      │ 有价值视角: ...         │
│ 6 台词DNA        │ X条      │ 高频词: ...             │
├──────────────────┼──────────┼──────────────────────────┤
│ 信息不足维度      │ 无/有     │ [列出需要用户补充的维度]  │
└──────────────────┴──────────┴──────────────────────────┘
```

用户确认调研质量OK → 进入Phase 2。
用户觉得某维度不够 → 补充调研后再继续。

### Phase 2.5 提炼确认检查点

Phase 2角色提炼完成后，必须暂停展示提炼摘要给用户确认：

```
提炼结果摘要：
- 角色核心特质：N个（列出名称）
- 行为模式：N条
- 台词DNA：[3个关键特征]
- 表演边界：N条
- 角色弧线：N个阶段
```

用户确认OK → 进入Phase 3构建。
用户觉得某个维度不够 → 回到Phase 2调整后再继续。

## 质量验证标准

### 必填项检查

| 检查项 | 标准 |
|--------|------|
| 核心特质数量 | 3-7个 |
| 行为模式数量 | 5-10条 |
| 表演边界数量 | 至少3条 |
| 关系动态数量 | 主要关系都有描述 |
| 台词DNA | 包含高频词、禁忌词、句式特征 |

### 质量门槛

- 台词测试：5句新台词中至少4句能识别出角色
- 场景测试：新场景下角色反应符合预期
- 边界测试：角色的边界行为有明确定义

## 更新协议

### 更新触发条件

- 用户主动要求更新
- 原著/影视有新版本发布
- 发现重大理解偏差

### 更新流程

1. 读取现有SKILL.md
2. 标注需要刷新的部分
3. 执行增量调研
4. 合并新旧信息
5. 输出更新后的SKILL.md

### 版本管理

- 每次更新在文件头添加版本记录
- 保留历史版本注释（被替换的内容）

## Agent调研规范

### 信息来源优先级

1. 经典文学（四大名著）：原著文本优先，影视改编作为补充
2. 影视角色：原片/剧本优先，表演分析作为补充
3. 动漫角色：原作漫画/动画优先，声优表演作为参考

### Agent输出要求

每个Agent必须：
1. 将调研结果写入对应md文件
2. 标注信息来源
3. 提供原始证据（引用原文/台词）

### 禁止事项

- 不允许只做调研不存文件
- 不允许将文件存到skill目录外部
- 不允许使用未标注来源的信息

## 违规处理

### 常见问题

| 问题 | 处理方式 |
|------|---------|
| 调研文件缺失 | 补全后重新合成 |
| 角色特质过少 | 补充至少3个核心特质 |
| 边界定义模糊 | 明确至少3条具体边界 |
| 信息来源不明 | 标注来源或删除该信息 |
